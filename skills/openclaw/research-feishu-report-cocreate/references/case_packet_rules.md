# Case Packet Rules

Use this module whenever an abstract mechanism or claim needs a concrete production example.

## Case Packet Fields

- actor
- scene
- trigger
- workflow
- data_or_object
- decision
- failure_mode
- reuse_condition

## Standard

- choose cases that match the target audience's world
- prefer detailed, real production cases over polished anecdotes
- use cases to clarify mechanism, not just decorate the report

## Minimum

- heavy reports should include multiple concrete case packets
- every difficult concept should have at least one concrete case nearby

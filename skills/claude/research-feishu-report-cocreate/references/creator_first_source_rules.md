# Creator-First Source Rules

Use this module when the report quality depends on original information depth.

## Source Hierarchy

Prefer sources in this order:

1. official repos, docs, specs, papers, release notes, code, commits
2. creator-authored or creator-supported artifacts such as talks, issue threads, examples, engineering posts
3. direct practitioner evidence and verifiable product behavior
4. secondary analysis only as context, never as the main proof

## Required Practice

- Create an atomic evidence ledger before making strong claims.
- Tag each evidence object with source type and what it directly proves.
- If the best available proof is secondary or indirect, lower confidence explicitly.
- Do not let media summaries stand in for creator knowledge.

## Minimum Standard

- heavy reports should gather enough primary evidence to support all major claims
- major claims should reference at least two evidence objects where possible
- the report should distinguish proved, inferred, and unknown

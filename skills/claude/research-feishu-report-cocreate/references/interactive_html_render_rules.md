# Interactive HTML Render Rules

Use this module only when the report has already passed the content-quality gates and an interactive HTML output is explicitly required.

## Purpose

Turn a finalist report into a higher-density, easier-to-explore reading surface.

## Required Capabilities

- clear thesis-first landing
- expandable evidence and counter-case sections
- case-card drilldown
- figure interactions that clarify reasoning, not just animate it

## Rules

- do not simply restyle the Feishu output
- optimize for scannability and drill-down
- keep desktop and mobile readability acceptable
- preserve the same core judgment as the canonical dossier
